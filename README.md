## Overview

This repository contains open-source simulation code, figures, references, and both IEEE and ACM conference paper versions accompanying the research paper:

> **"Entangled Intelligence: Nanoscale Quantum-Neuromorphic Hybrid Architectures for Post-von Neumann Computation"**  
> Sunil Gentyala  
> 26th IEEE International Conference on Nanotechnology (IEEE-NANO 2026), July 5-8, 2026, Nanjing, China

The paper proposes and analyzes the **Quantum-Neuromorphic Hybrid Substrate (QNHS)**: a vertically integrated monolithic architecture co-integrating:
- **28Si spin qubit arrays** (Quantum Coherent Processing Plane, QCPP)
- **PMA-MTJ spintronic memristors** (Spintronic Synaptic Plane, SSP)
- **3 nm cryo-CMOS FinFET circuits** (CMOS Peripheral Plane, CPP)

The architecture projects **~11 fJ per synaptic event** (three orders of magnitude below GPU baselines) with intrinsic Bayesian uncertainty quantification and hardware-native security primitives.

---

## Key Contributions

1. **QNHS Three-Plane Architecture**: Monolithic vertical integration of quantum, spintronic, and CMOS planes bonded via Cu-Cu thermocompression at 200 nm pitch.

2. **Quantum Synaptic Encoding**: Each synapse encoded as a k-qubit quantum state in a 2^k-dimensional Hilbert space, providing intrinsic Bayesian weight distributions.

3. **Q-STDP Learning Rule**: Quantum amplitude-domain spike-timing-dependent plasticity that reproduces Hebbian learning while encoding probabilistic weight distributions.

4. **Distance-3 Surface Code Protection**: Topological error correction at 1 K projecting 100 ms logical coherence lifetimes across multiple spike integration windows.

5. **Security Analysis**: First comprehensive threat model for quantum-neuromorphic hardware, including side-channel attacks (GLSVLSI 2025, NDSS 2025), adversarial weight manipulation, and post-quantum cryptography acceleration via the MTJ crossbar.

6. **Hardware Security Primitives**: Physical unclonable functions (QD-PUFs), true random number generation (MTJ-QRNG), and PQC-accelerating analog matrix arithmetic.

## Simulation Modules

### 1. Energy Model (`simulations/energy_model.py`)
Implements the three-component synaptic energy decomposition:
- E_MTJ from Landauer-Buttiker spin-torque switching model
- E_qubit from gate drive power and pi-pulse duration
- E_CMOS from cryo-CMOS peripheral overhead at 1 K

**Run:** `python simulations/energy_model.py`

### 2. Q-STDP Simulation (`simulations/qstdp_simulation.py`)
Simulates quantum spike-timing-dependent plasticity on a single k-qubit synapse:
- Amplitude update under the Q-STDP rule (Eq. 3 in paper)
- Weight distribution evolution and entropy tracking
- Bayesian uncertainty quantification in the quantum amplitude space

**Run:** `python simulations/qstdp_simulation.py`

### 3. MTJ Resistance Model (`simulations/mtj_resistance_model.py`)
Models PMA-MTJ device characteristics:
- Four resistance states {R0, R1, R2, R3} with TMR > 200%
- Thermally activated switching probability (Neel-Brown model)
- Crossbar weight state visualization

**Run:** `python simulations/mtj_resistance_model.py`

### 4. Surface Code Analysis (`simulations/surface_code_analysis.py`)
Analyzes distance-3 surface code error suppression:
- Logical error rate vs. code distance
- Coherence time enhancement factor
- Verification that logical T2 > spike integration window

**Run:** `python simulations/surface_code_analysis.py`

## Security Contributions

This paper provides the first security analysis for a quantum-neuromorphic hardware substrate. Key findings:

- **Side-channel threats**: Timing attacks on cloud quantum services (GLSVLSI 2025, DOI: 10.1145/3716368.3735264) and crosstalk-based circuit reconstruction (NDSS 2025, arXiv:2412.10507) motivate hardware-level countermeasures.
- **Adversarial robustness**: Quantum amplitude encoding reduces gradient-based adversarial attack effectiveness; training-phase Q-STDP poisoning requires differential privacy countermeasures.
- **Native primitives**: QD-PUFs for device authentication, MTJ-QRNG for cryptographic key seeding, crossbar PQC acceleration for NIST FIPS 203/204/205 lattice schemes.

Full threat model: [security/threat_model.md](security/threat_model.md)
